
** 실행 방법 **

DotSurvival_Prototype\Build 폴더 안의 New Unity Project.exe 파일을 실행하면 게임을 실행할 수 있습니다.

방 만들기:
	1. local 버튼 클릭
	2. 닉네임 입력, create room 버튼 클릭
	3. 플레이어 수 선택, 선택하지 않으면 8로 자동 설정 (방 제목 입력은 틀만 있고 아직 미구현), confirm 버튼 클릭

방 찾기, 참가:
	1. local 버튼 클릭
	2. 닉네임 입력, find room 버튼 클릭
	3 좌 상단에 뜬 버튼을 클릭하여 방에 참가.

게임 시작: 모든 플레이어들이 ready 버튼을 누른 상태가 되면 시작.


** 조작법 **

W,A,S,D	: 상하좌우 이동
마우스 	: 마우스 위치로 조준, 조준한 지점을 총기가 바라봄.
좌클릭 	: 총알 발사
우클릭 	: 대쉬
F키	: 아이템 줍기
E키	: 보유한 아이템 정보
Tab키	: 순위표
1,2,3,4	: 보유한 아이템 버리기

** 주의점 **

1.마우스 커서

-현재 게임 플레이 씬에서 마우스 커서가 안 보입니다.
게임 플레이 씬에 진입할 때 조준점을 보여주기 위해 커서를 숨기기 때문입니다.

-E키를 누르면 아이템 정보 창이 뜨는데, 이때 마우스 커서가 보여지도록 설정해 두었습니다.
그 외의 경우는 아직 처리가 되어있지 않아,
게임 플레이 룸에서 그냥 벗어나면 커서가 사라진 것 처럼 보일 수 있습니다.
만약 커서가 사라진 것 처럼 보인다면 작업관리자를 켜 주면 됩니다.

2.순위표 갱신

-호스트에선 순위 변동이 생길 시 정상적으로 곧바로 순위표가 갱신됩니다.
하지만 클라이언트에선 여러가지 변수들이 네트워크 상으로 동기화 작업이 완료되기 전에 순위표 갱신을 하여
순위표가 곧바로 정상 갱신되지 않을 수 있습니다.
이것은 성능을 위해 순위표를 실시간 갱신하지 않고 있기 때문에 발생하는 현상 입니다.

-이것에 대한 현상은 클라이언트에서 순위표를 켜 둔 채로 순위표 갱신이 발생하면 플레이어 킬, 데스 등 
동기화가 필요한 작업들이 반영이 안 된 순위표를 보여주는 현상이 일어나게 되는데,

이런 경우, Tab 키를 두번 눌러 다시 켜 주면  해결됩니다.

( * 다만 순위표 갱신과 순위 계산은 다른 함수에서 실행되고,
    순위 계산은 호스트 권한이기 때문에 게임 룰을 위반하는 경우는 생기지 않습니다.(
( * 
	순위 계산은 호스트에서 플레이어들의 킬과 데스를 통해 계산하고,
	순위표 구성은 각자의 컴퓨터에서 각자의 변수들을 통해 만들어 내기 때문에,
	그 변수들의 동기화가 완료되기 전에 순위표를 구성해 발생하는 약한 버그.
	순위표 구성을 변수 동기화 완료 후에 실행되도록 보장할 수 있다면 해결될 것으로 보임.
)
